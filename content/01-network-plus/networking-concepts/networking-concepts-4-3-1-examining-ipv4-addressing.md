Learner Objective: *Identify and explain the process and components of IPv4 addressing, given a scenario.*    

Description: In this episode, the learner will examine IPv4 addressing in TCP/IP networks. We will explore the process and components that make up an IPv4 address.

--------  

* What do we mean by IPv4 addresses?
  + Unique identifiers for devices on a network.
  + 32 bits divided into four octets, like 192.168.1.1.
  + Each of the four octets in the address can have a value from 0 to 255, providing a range of possible addresses.
* Why is understanding binary important for IP addressing and subnetting?
  + Binary is crucial because IP addresses, though often represented in decimal, are fundamentally processed in binary by network devices.
  + This knowledge is key to understanding network addressing and how subnetting works at a technical level.
* What role does a subnet mask play?
  + A subnet mask works alongside an IP address to define which part of the address identified or belongs to the network and which part refers to the host.
  + Determine the subnet to which an IP address belongs.
* How do we tell what network we are on?
  + The network ID \(or network portion\) of an IP address, determined by the subnet mask, indicates the specific network a device is on.
  + The host ID \(or node ID\), which is the part of the IP address not covered by the subnet mask, identifies the specific device within that network.
  + Understanding the network and host portions of an IP address is crucial for determining both the network a device belongs to and its unique identifier within that network.

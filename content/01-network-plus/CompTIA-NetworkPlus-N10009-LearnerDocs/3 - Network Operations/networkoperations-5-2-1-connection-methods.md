## Connection Methods
At the end of this episode, I will be able to:    

1. Discuss common methods of secure remote access

Learner Objective: *Learn about various connection methods that can be made to access remote resources*    

Description: In this episode, you will learn about various connection methods for access to remote resources. These connection methods include console access, GUIs, APIs, and SSH.  

-----------

* Console access to remote resources refers to the ability to manage and control computing devices, servers, or network equipment located in distant locations using a text-based interface. This capability is essential for administrators and support personnel to perform troubleshooting, configuration, and maintenance tasks on remote systems as if they were physically present, ensuring efficient management and operation of distributed infrastructure.
* GUI access to remote resources involves using a Graphical User Interface to interact with and control computing devices, servers, or applications located in distant locations. This allows users to navigate and manage remote systems through a visual interface, facilitating tasks such as software configuration, file management, and monitoring as if the resources were locally accessible.
* API access to remote resources involves using Application Programming Interfaces to interact programmatically with computing devices, servers, or services located in distant locations. This enables developers to automate tasks, retrieve data, or trigger actions on remote systems through a standardized set of commands or requests, streamlining integration and enhancing the efficiency of distributed applications and services.
* SSH (Secure Shell) access for remote resources provides a secure and encrypted method for accessing and managing computing devices or servers over a network. Users can establish a command-line interface connection to the remote system, allowing for secure execution of commands, file transfers, and administration tasks with confidentiality and integrity.
